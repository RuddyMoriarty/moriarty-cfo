# Data Room — Checklist de constitution

Documents à préparer pour la due diligence d'une levée de fonds ou cession.

## 01. Documents juridiques

- [ ] Statuts à jour
- [ ] Kbis < 3 mois
- [ ] Pacte d'actionnaires
- [ ] PV des AG des 3 dernières années
- [ ] PV des conseils d'administration
- [ ] Conventions réglementées (art. L. 225-38 / L. 223-19)
- [ ] Bénéficiaires effectifs (registre RBE)
- [ ] Licences / autorisations (sectorielles)

## 02. Financials

- [ ] États financiers audités N-3, N-2, N-1
- [ ] Liasses fiscales N-3, N-2, N-1
- [ ] Annexe complète
- [ ] Balance + grand livre
- [ ] FEC
- [ ] Reporting mensuel détaillé 24 derniers mois
- [ ] Forecast 12 mois
- [ ] Business plan 3-5 ans
- [ ] Politique comptable (méthodes amortissements, provisions)
- [ ] Tableau emprunts + tableau d'amortissements
- [ ] Rapport CAC (si applicable)
- [ ] Rapport spécial CAC (conventions réglementées)

## 03. Fiscal

- [ ] Liasses fiscales 3 ans
- [ ] FEC 3 ans
- [ ] Déclarations TVA 24 derniers mois
- [ ] DSN 24 derniers mois
- [ ] CIR / CII : déclarations + dossier justificatif
- [ ] Avis d'impôts (CFE, CVAE)
- [ ] Avis URSSAF
- [ ] Procédures fiscales / sociales en cours

## 04. Commercial

- [ ] Top 10 contrats clients (signés, en cours)
- [ ] Pipeline commercial pondéré
- [ ] Politique tarifaire
- [ ] CGV à jour
- [ ] Top 10 fournisseurs (contrats, conditions)
- [ ] Données KPIs (ARR, MRR, churn, NPS, LTV/CAC)
- [ ] Études marché récentes
- [ ] Brevets / marques

## 05. Produit / IP

- [ ] Documentation produit
- [ ] Roadmap produit 12-24 mois
- [ ] Brevets déposés (et lettres de cession des inventeurs)
- [ ] Marques déposées
- [ ] Code source : structure, lignes, dette technique
- [ ] Politique de propriété intellectuelle
- [ ] Contrats licence (logiciels open source utilisés)

## 06. RH

- [ ] Organigramme
- [ ] Effectif détaillé (CDI, CDD, alternance, freelances)
- [ ] Contrats top 10 collaborateurs (CDI, conventions, packages)
- [ ] Politique salariale (grille, NAO)
- [ ] Plans BSPCE / AGA (attribution, vesting, exercices)
- [ ] Convention collective applicable
- [ ] Litiges sociaux en cours
- [ ] Politique formation (plan, FNE)
- [ ] CSE : composition, PV
- [ ] Politique télétravail / flex office

## 07. Litiges et risques

- [ ] Litiges en cours (liste exhaustive : avocats, montants, statuts)
- [ ] Litiges passés réglés (3 dernières années)
- [ ] Réclamations clients / fournisseurs significatives
- [ ] Procédures réglementaires (URSSAF, DGFiP, CNIL, ACPR si applicable)
- [ ] Cyber-incidents passés

## 08. Tech infrastructure

- [ ] Architecture technique (schéma)
- [ ] Hébergement (AWS, GCP, Azure : régions, tiers, contrats)
- [ ] Sécurité IT : politique, certifications (ISO 27001, SOC 2)
- [ ] Sauvegardes / DRP
- [ ] Politique BYOD / télétravail sécurisé
- [ ] Inventaire matériel
- [ ] Stack technique (langages, frameworks, outils)
- [ ] Politique mises à jour / patching

## 09. Compliance

- [ ] Politique RGPD (registre traitements, DPO, breach notification)
- [ ] LCB-FT (si applicable)
- [ ] Politiques internes (anti-corruption, lanceur d'alerte)
- [ ] Audits sécurité / pen-test récents
- [ ] Cyber-assurance (police)
- [ ] RC pro (police)
- [ ] D&O (police)
- [ ] Multirisque (police)

## 10. Immobilier / actifs

- [ ] Liste des locaux loués (baux, durée, loyers)
- [ ] Liste des locaux possédés (titres de propriété)
- [ ] Liste équipements significatifs (CAPEX > 10 k€)
- [ ] Crédits-bail en cours

## 11. Données opérationnelles (selon secteur)

- [ ] **SaaS** : MRR / ARR détaillés, cohorte, churn, expansion
- [ ] **E-commerce** : panier moyen, fréquence, taux conversion, retour
- [ ] **Industrie** : volume production, taux qualité, CAPEX
- [ ] **Services** : utilisation collaborateurs, TJM, marges projet

## 12. Acquisitions précédentes (si applicable)

Pour chaque acquisition :
- [ ] SPA + amendments
- [ ] Working capital adjustment
- [ ] Earn-out (si en cours)
- [ ] Garanties de passif
- [ ] Plan d'intégration + résultats

| **Sharepoint / Notion** | OK pour amorces (BA, seed) |

## Bonnes pratiques

- **Index clair** par section et sous-section
- **Permissions granulaires** (visualisation, téléchargement)
- **Tracking** : qui voit quoi, quand
- **Mise à jour continue** : ne pas laisser de docs obsolètes
- **Sécurité** : pas de mots de passe / clés API dans la data room
- **Pre-due diligence interne** : faire revoir par votre avocat avant ouverture aux investisseurs
